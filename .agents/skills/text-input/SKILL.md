# 文本输入操作

## 核心原则

**输入前必须确认输入法已打开**，否则输入无效。

## 输入法状态判断

### 确认输入法已打开
屏幕底部出现 **"ADB Keyboard {ON}"** 字样 = 输入法已激活，可以输入。

### 输入法未打开时
如果看不到 "ADB Keyboard {ON}"：
1. 点击输入框尝试激活
2. 等待 1 秒后再次检查
3. 确认输入法激活后再进行输入

## 清空输入框

当需要输入全新内容时，必须先清空原有内容：

### 方法一：点击清除图标（优先）
检查输入框**右侧**是否有 **x 图标**（清除按钮）：
- 有 x 图标 → 点击该图标，快速清空内容
- 无 x 图标 → 使用方法二

### 方法二：连续删除
当输入框没有清除图标时，需要判断原有内容的长度，然后**连续调用多次 delete**：

```
原有内容长度估算 → delete 调用次数
1-2 个字          → 2 次
3-5 个字          → 5 次
6-10 个字         → 10 次
超过 10 个字      → 15 次（宁多勿少）
```

**注意**：delete 调用次数应略多于字数，确保完全清空。

## 操作流程

当需要输入文本时：

```
1. [截屏] 查看当前状态
2. [检查] 屏幕底部是否有 "ADB Keyboard {ON}"
   ├─ 有 → 输入法已激活，继续下一步
   └─ 无 → 点击输入框，等待后再次确认
3. [判断] 输入框是否有内容需要清空
   ├─ 有内容 + 有 x 图标 → 点击 x 图标清空
   ├─ 有内容 + 无 x 图标 → 连续调用 delete 清空
   └─ 无内容 → 直接输入
4. [输入] 使用 type_text 输入新内容
```

## 示例

### 示例一：清空后输入（有 x 图标）
```
User: 搜索"张三"
1. [截屏] 看到搜索框内有 "test"，右侧有 x 图标
2. [检查] 底部有 "ADB Keyboard {ON}"
3. [清空] 点击 x 图标，内容已清空
4. [输入] type_text("张三")
```

### 示例二：清空后输入（无 x 图标）
```
User: 把搜索内容改成"abc"
1. [截屏] 看到搜索框内有 "test"（4个字），无 x 图标
2. [检查] 底部有 "ADB Keyboard {ON}"
3. [清空] 连续调用 5 次 delete
4. [截屏] 确认内容已清空
5. [输入] type_text("abc")
```

### 示例三：直接输入
```
User: 在搜索框输入"hello"
1. [截屏] 看到搜索框为空
2. [检查] 底部有 "ADB Keyboard {ON}"
3. [输入] type_text("hello")
```

## 常见错误

| 错误行为 | 正确做法 |
|---------|---------|
| 未确认输入法状态就输入 | 先检查 "ADB Keyboard {ON}" |
| 直接 type_text 覆盖原内容 | 先清空再输入 |
| delete 次数不足 | 宁多勿少，确保清空 |
| 忽略 x 图标 | 优先使用 x 图标清空 |
