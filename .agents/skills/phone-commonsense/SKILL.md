## 分页与页面指示器

### 识别分页指示器

分页指示器通常显示为：
- **圆点样式**：
  - 实心/空心：实心圆点表示当前页，空心圆点表示其他页
  - 大/小：大圆点表示当前页，小圆点表示其他页
  - 颜色深浅：深色圆点表示当前页，浅色圆点表示其他页
- **横线样式**：
  - 长/短：长横线表示当前页，短横线表示其他页
  - 实心/空心：实心横线表示当前页，空心横线表示其他页
  - 颜色深浅：深色横线表示当前页，浅色横线表示其他页
- **数字**：1/5、2/5 等
- **位置**：通常在屏幕底部或内容区域底部

### 分页遍历原则

```
[错误做法]：
左滑一下 → 没找到 → 右滑一下 → 没找到 → 结论：不存在
（这只查看了2页，且可能来回重复）

[正确做法]：
1. [观察] 数一下分页指示器有多少个点/页
2. [记录] 当前在第几页（实心点的位置）
3. [遍历] 从当前页开始，单向滑动直到查看完所有页
   - 假设有4页，当前在第1页：第1页 → 第2页 → 第3页 → 第4页
   - 假设有4页，当前在第3页：第3页 → 第4页 → 返回第1页 → 第2页
4. [确认] 所有页面都查看完毕后才能下结论
```

### 分页遍历示例

```
任务：在桌面找到"微信"应用
1. [截屏] 看到底部有5个分页点（当前第1页）
2. [遍历] 需要查看5页
   - 第1页：未找到 → 左滑
   - 第2页：未找到 → 左滑
   - 第3页：未找到 → 左滑
   - 第4页：未找到 → 左滑
   - 第5页：未找到
3. [结论] 5页都查看完毕，桌面没有"微信"应用
```

## 列表与滚动

### 列表类型判断

| 类型 | 特征 | 遍历方式 |
|-----|------|---------|
| 分页列表 | 底部有分页指示器 | 按页遍历 |
| 无限列表 | 滚动加载更多 | 滚动到底部判断 |
| 固定列表 | 内容固定，无加载更多 | 滚动查看全部 |

### 判断是否滚动到底部

**底部特征**：
- 出现"没有更多了"、"已经到底了"、"END"等文字
- 最后一条内容重复出现（无法继续滚动）
- 滚动后内容不再变化

### 列表遍历流程

```
1. [截屏] 查看当前列表状态
2. [判断] 列表类型
   ├─ 分页列表 → 按分页遍历原则处理
   └─ 滚动列表 → 执行以下流程
3. [记录] 记住当前可见的最后一条内容
4. [滚动] 向上滑动一屏
5. [对比] 新内容与记录的最后一条是否相同
   ├─ 相同 → 已到底部
   └─ 不同 → 重复步骤3-5
```

## 避免死循环

### 常见死循环场景

| 场景 | 错误行为 | 正确做法 |
|-----|---------|---------|
| 查找元素 | 左滑→右滑→左滑→右滑 | 单向遍历所有页面 |
| 滚动列表 | 上下反复滚动 | 持续向下直到到底 |
| 等待加载 | 无限等待 | 设置超时，超时后报错 |
| 点击失败 | 反复点击同一位置 | 截屏确认状态，调整策略 |

### 防死循环原则

```
1. 每次操作后必须截屏确认结果
2. 记录已执行的操作和查看的位置
3. 设定最大尝试次数（如：滑动不超过10次）
4. 状态未变化时停止重复操作
5. 无法继续时主动报错，不要无限尝试
```

## 屏幕状态判断

### 加载状态

**特征**：
- 转圈动画（加载中...）
- 骨架屏（灰色占位块）
- 进度条

**处理**：等待加载完成后再操作

### 空状态

**特征**：
- 空白图标 + 提示文字
- "暂无数据"、"没有内容"等
- 引导操作的按钮

**处理**：确认是空状态，不需要继续查找

### 错误状态

**特征**：
- 错误图标 + 提示文字
- "加载失败"、"网络错误"等
- "重试"按钮

**处理**：可尝试点击重试，或报告错误

### 弹窗与对话框

**处理原则**：
- 优先处理弹窗（可能阻挡操作）
- 阅读弹窗内容，选择合适操作

## 常见UI元素位置

### 固定位置元素

| 元素 | 典型位置 | 作用 |
|-----|---------|------|
| 返回按钮 | 左上角 | 返回上一页 |
| 关闭按钮 | 右上角（X图标） | 关闭当前页面/弹窗 |
| 确认按钮 | 弹窗右下角 | 确认操作 |
| 取消按钮 | 弹窗左下角 | 取消操作 |
| 搜索按钮 | 右上角（放大镜） | 打开搜索 |
| 更多按钮 | 右上角（三点或三横） | 展开菜单 |
| 底部导航栏 | 屏幕最底部 | 切换主要功能模块 |
| 悬浮按钮 | 右下角 | 快捷操作（如新建） |

### 滚动区域判断

- **顶部固定**：标题栏、搜索框通常固定不滚动
- **底部固定**：导航栏、操作按钮通常固定不滚动
- **中间滚动**：内容区域可滚动

## 操作时机判断

### 何时等待

- 页面正在加载（转圈）
- 动画正在播放
- 键盘正在弹出
- 内容正在刷新

### 何时可以操作

- 加载动画消失
- 内容完全显示
- 按钮可点击状态
- 无弹窗遮挡

### 操作间隔

```
点击后 → 等待 0.5-1 秒（响应时间）
滑动后 → 等待 0.3-0.5 秒（动画时间）
输入后 → 等待 0.2-0.3 秒（输入响应）
长按后 → 等待 0.5 秒（菜单弹出）
```

## 下拉刷新

### 识别下拉刷新

- 列表顶部出现刷新指示器
- "下拉刷新"、"释放刷新"文字
- 转圈动画

### 执行下拉刷新

```
从屏幕顶部稍下位置开始，向下滑动：
swipe(500, 100, 500, 400)
```

**用途**：获取最新数据、解决加载异常

## 常见错误

| 错误行为 | 后果 | 正确做法 |
|---------|------|---------|
| 不看分页指示器 | 遗漏内容 | 先数页数，再遍历 |
| 来回滑动查找 | 死循环、遗漏 | 单向遍历所有页 |
| 不判断列表底部 | 无限滚动 | 检测到底标志 |
| 忽略加载状态 | 操作无效 | 等待加载完成 |
| 反复点击失败点 | 死循环 | 截屏分析原因 |
| 不记录已查看内容 | 重复查看 | 记住已查看的位置 |
