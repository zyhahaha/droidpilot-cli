# 手机手势操作

## 核心原则

**理解归一化坐标系统是正确操作手机的基础**。

## 坐标系统

### 归一化坐标系定义

使用 **0-1000 范围** 的归一化坐标，与屏幕实际分辨率无关：

```
(0,0) ──────────────► X轴（水平方向）
  │
  │
  │        (500,500)
  │          中心点
  │
  ▼
Y轴（垂直方向）
                          (1000,1000)
                            右下角
```

### 关键坐标点

| 位置 | 坐标 |
|-----|------|
| 左上角 | (0, 0) |
| 右上角 | (1000, 0) |
| 左下角 | (0, 1000) |
| 右下角 | (1000, 1000) |
| **屏幕中心** | **(500, 500)** |
| 水平中线上方 | (500, 250) |
| 水平中线下方 | (500, 750) |
| 垂直中线左侧 | (250, 500) |
| 垂直中线右侧 | (750, 500) |

### 坐标规则

- **X轴**：水平方向，0=最左边，1000=最右边
- **Y轴**：垂直方向，0=最上边，1000=最下边
- **坐标格式**：`(x, y)`，范围均为 0-1000

## 基础手势操作

### 点击 (tap_screen)

点击指定坐标位置：

```
tap_screen(x, y)
```

**用途**：点击按钮、链接、图标等可交互元素

### 长按 (long_press)

长按指定坐标位置：

```
long_press(x, y, duration=1000)
```

**用途**：弹出菜单、进入编辑模式、拖拽准备

### 滑动 (swipe)

滑动操作由 **起点** 滑动到 **终点**：

```
swipe(start_x, start_y, end_x, end_y, duration=300)
```

## 滑动方向详解

### 左右滑动（水平滑动）

**Y轴不变，X轴变化**

#### 向右滑动（从左往右滑）
```
swipe(200, y, 800, y)
        ↑        ↑
      起点X     终点X（更大）
      
Y坐标相同，X坐标增大
```
**效果**：页面内容向右移动（如：查看左侧内容、返回上一页）

#### 向左滑动（从右往左滑）
```
swipe(800, y, 200, y)
        ↑        ↑
      起点X     终点X（更小）
      
Y坐标相同，X坐标减小
```
**效果**：页面内容向左移动（如：查看右侧内容、切换下一个）

### 上下滑动（垂直滑动）

**X轴不变，Y轴变化**

#### 向下滑动（从上往下滑）
```
swipe(x, 200, x, 800)
        ↑        ↑
      起点Y     终点Y（更大）
      
X坐标相同，Y坐标增大
```
**效果**：页面内容向下移动（如：查看上方内容、向上翻页）

#### 向上滑动（从下往上滑）
```
swipe(x, 800, x, 200)
        ↑        ↑
      起点Y     终点Y（更小）
      
X坐标相同，Y坐标减小
```
**效果**：页面内容向上移动（如：查看下方内容、向下翻页、刷新）

## 滑动方向速查表

| 操作意图 | 手势方向 | 坐标变化 | 示例（Y=500为中线） |
|---------|---------|---------|------|
| 查看右侧内容 | 从右往左滑 | X减小，Y不变 | swipe(800, 500, 200, 500) |
| 查看左侧内容 | 从左往右滑 | X增大，Y不变 | swipe(200, 500, 800, 500) |
| 查看下方内容 | 从下往上滑 | Y减小，X不变 | swipe(500, 800, 500, 200) |
| 查看上方内容 | 从上往下滑 | Y增大，X不变 | swipe(500, 200, 500, 800) |

## 操作流程

当需要进行屏幕操作时：

```
1. [截屏] 查看当前屏幕状态
2. [定位] 确定目标元素的归一化坐标位置
3. [判断] 需要执行的操作类型
   ├─ 点击元素 → tap_screen(x, y)
   ├─ 长按元素 → long_press(x, y)
   ├─ 滚动页面 → 根据方向选择滑动
   │   ├─ 向下翻页 → swipe(500, 800, 500, 200)
   │   ├─ 向上翻页 → swipe(500, 200, 500, 800)
   │   ├─ 向左翻页 → swipe(800, 500, 200, 500)
   │   └─ 向右翻页 → swipe(200, 500, 800, 500)
   └─ 滑动元素 → 确定起点终点后执行 swipe
4. [验证] 截屏确认操作结果
```

## 示例

### 示例一：点击按钮
```
User: 点击"登录"按钮
1. [截屏] 看到登录按钮位于屏幕中下方
2. [定位] 按钮坐标约为 (500, 750)
3. [操作] tap_screen(500, 750)
```

### 示例二：向下翻页查看更多内容
```
User: 向下滚动查看更多
1. [截屏] 当前显示列表顶部内容
2. [操作] swipe(500, 800, 500, 200)
                ↑           ↑
              起点Y高     终点Y低（向上滑）
3. [效果] 页面向上滚动，显示下方内容
```

### 示例三：向左滑动切换页面
```
User: 切换到下一个标签页
1. [截屏] 当前在第一个标签页
2. [操作] swipe(800, 500, 200, 500)
                ↑         ↑
              起点X大   终点X小（向左滑）
3. [效果] 页面向左切换，显示右侧内容
```

### 示例四：长按弹出菜单
```
User: 长按消息进行删除
1. [截屏] 看到目标消息
2. [定位] 消息坐标约为 (500, 400)
3. [操作] long_press(500, 400, 1000)
4. [截屏] 确认弹出操作菜单
5. [操作] 点击"删除"选项
```

### 示例五：点击屏幕角落
```
User: 点击右上角的设置图标
1. [截屏] 看到设置图标在右上角
2. [定位] 坐标约为 (900, 50)
3. [操作] tap_screen(900, 50)
```

## 常见错误

| 错误行为 | 正确做法 |
|---------|---------|
| 混淆滑动方向 | 记住：手指滑动方向 = 内容移动的反方向 |
| X/Y坐标搞反 | X是水平，Y是垂直，格式永远是(x, y) |
| 使用像素坐标 | 必须使用归一化坐标（0-1000） |
| 滑动距离太短 | 滑动距离应足够（建议400-600单位） |
| 滑动速度太快 | 添加适当duration（默认300ms） |
| 点击坐标不准确 | 根据截屏仔细定位元素中心点 |

## 坐标估算技巧

使用归一化坐标时的常用位置参考：

```
屏幕中心：(500, 500)

区域划分：
┌──────────┬──────────┐
│          │          │
│  左上     │  右上     │
│ (250,250)│ (750,250)│
├──────────┼──────────┤
│  左中     │  右中     │
│ (250,500)│ (750,500)│
├──────────┼──────────┤
│  左下     │  右下     │
│ (250,750)│ (750,750)│
└──────────┴──────────┘

常用位置速查：
- 顶部边缘：y ≈ 0-100
- 底部边缘：y ≈ 900-1000
- 左侧边缘：x ≈ 0-100
- 右侧边缘：x ≈ 900-1000
- 上半屏：y ≈ 250
- 下半屏：y ≈ 750
- 左半屏：x ≈ 250
- 右半屏：x ≈ 750
```

## 归一化坐标的优势

1. **屏幕无关**：适配任意分辨率的设备
2. **易于估算**：中心点永远是(500, 500)
3. **简单直观**：0-1000范围便于心算
4. **一致性**：相同操作在任何设备上使用相同坐标
